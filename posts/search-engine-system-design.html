Search Engine System Design

1. Data Collection: structured or unstructured

Data collection can include:

  • - web crawlers
  • - user inputs
  • - APIs
  • - external databases

2. Indexing: create a document object

Break down the document into more manageable parts. Identify key elements. Create a structured representation using unstructured data. The goal is to match the most relevant documents to the query.

Document object can include:

  • - document classification / category mapping
  • - document annotation / tagging
  • - document rewriting
  • - document moderation
  • - document-independent ranking features / signals

3. Query Parser: create a query object

Break down the query into its components. Analyze the meaning behind the words. Determine the user's intent. The goal is to match the query to the most relevant documents.

Query object can include:

  • - query classification / category mapping
  • - query scoping / tagging
  • - query rewriting
  • - query-independent ranking features / signals

4. Personalization Engine: create a user object

Analyze patterns in user behavior. Consider past interactions. Predict what a user might be looking for. The goal is to match the user to the most relevant documents.

User object can include:

  • - user classification / category mapping
  • - user annotation / tagging
  • - user-independent ranking features / signals

5. Information Retrieval (IR): match queries to documents

Pull relevant documents using the document objects and query object as inputs to a relevance function. The goal is to put together an initial ranked list of the most relevant documents.

IR can include:

  • - text based retrieval
  • - embedding based retrieval

6. Ranking: order matched documents by relevance

Predict a relevance score using features for each query|document pair as inputs to a ranking model. Sort those documents according to their scores. The goal is to order the documents so that the most relevant documents are shown at the top of the list.

Ranking includes:

  • - ranking features: query-independent, document-independent, query-dependent
  • - ranking model

7. User Interface (UI): display the search results to the user

UI can include:

  • - title
  • - description
  • - sitelinks
  • - badges / labels
  • - reviews / ratings
